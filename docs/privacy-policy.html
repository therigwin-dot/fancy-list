Fancy List Web Part - Privacy Policy

Effective Date: July 2025

Developer: Therigwin's Fox Studios

Contact: therigwin+FoxShrineStudios@gmail.com

1. Information Collection

The Fancy List Web Part operates within your SharePoint environment and does not collect, store, or transmit any personal data outside of your organization's SharePoint infrastructure.

Data Processing:

  • Local Processing: All data processing occurs within your SharePoint environment
  • No External Storage: No data is transmitted to external servers or third-party services
  • SharePoint Integration: The web part only accesses data that users already have permission to view
  • Configuration Storage: Web part configuration settings are stored within SharePoint's standard configuration system

2. Data Access and Permissions

The web part respects SharePoint's existing permission model:

  • Permission Inheritance: Users can only access data they already have permission to view
  • No Elevated Access: The web part does not grant additional permissions or access
  • Audit Compliance: All data access follows SharePoint's standard audit trails

3. Configuration and Settings

Web part configuration data is stored within SharePoint's standard configuration system:

  • Property Pane Settings: Styling, filtering, and display preferences
  • List Configuration: Selected lists, fields, and display options
  • User Preferences: Individual user settings and customizations

This configuration data is stored within your SharePoint environment and is not accessible to external parties.

4. Third-Party Services

NO THIRD-PARTY SERVICES: The Fancy List Web Part does not integrate with any external services, APIs, or third-party platforms that could access your data.

5. Data Retention

Data retention follows your organization's SharePoint policies:

  • List Data: Retained according to your SharePoint list retention policies
  • Configuration: Stored within SharePoint's standard configuration system
  • User Settings: Follows SharePoint's user profile and settings retention

6. Security

The web part inherits SharePoint's security model:

  • Authentication: Uses SharePoint's standard authentication system
  • Authorization: Respects SharePoint's permission and access control
  • Data Protection: Benefits from SharePoint's built-in security features
  • Encryption: All data transmission uses SharePoint's encrypted connections

7. Updates and Changes

This privacy policy may be updated as the web part evolves. Users will be notified of significant changes through standard SharePoint communication channels.
